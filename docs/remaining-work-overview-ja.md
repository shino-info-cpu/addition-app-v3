# 残作業の見える化

2026-03-23 時点の、`v3` 再構築の残作業メモです。

## 数の見え方

- 試作候補枝: `24本`
- 仮置きなし: `3/24`
- 仮置きあり: `21/24`

上の `3/24` は「制度条件がいったん仮置きなしまで詰まっている枝」の数です。
ただし、実際の作業は枝1本ずつ完全独立ではなく、family 単位でまとまることが多いです。

## いま見えている残テーマ

細かい制度詰めフェーズとして、いま見えている残テーマは `8本` です。

1. 医保教 family
   通院同行の初回加算関係、情報共有の月区分境界、面談/会議の対象機関境界など。
2. 集中支援 family
   対象機関グループ境界、情報共有の制度境界、行為ラベルの制度語との差など。
3. モニタリング加算
   `サービス提供場面確認` と `提供現場訪問` の読み替え、対象機関境界。
4. 入院時情報連携 I / II
   `訪問情報提供 / 訪問以外情報提供` を今のUIでどう表現するか。
5. 保・教支援 family
   集団生活施設などの対象機関境界、`訪問面接 / 会議参加` の見せ方。
6. 居宅連携 family
   `訪問面接 / 会議参加` の見せ方、旧総称履歴の扱い。
7. 居宅連携（就労） family
   `訪問面接 / 会議参加` の見せ方、旧総称履歴の扱い。
8. 退院・退所加算
   `入所・入院期間中3回` の扱い、弱い名称の対象施設をどう拾うか。

## その後に残る大きい工程

制度詰めフェーズのあとに残る大きい工程は `2本` です。

1. `app.js` 直書きの試作枝を `addition` 系テーブルへ移す
2. 新加算を設定追加で増やせる形へする

## 今の見立て

- いま進めている「細かい工程」は、無限に続く段階ではない
- ただし `24本中3本完了` なので、制度詰めフェーズ自体はまだ前半から中盤
- 先に周辺機能を増やすより、当面はこの `8本` を優先するほうが芯がぶれにくい
