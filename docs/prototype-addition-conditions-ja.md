# 試作7加算の条件整理

この資料は、現在の [app.js](J:/マイドライブ/加算アプリV2/v3/app/frontend/app.js) に仮実装している7本の試作枝だけを整理したものです。

注意:

- 実際の加算種類が7つしかない、という意味ではありません。
- 新加算を設定追加だけで増やせる仕組みは、まだ未実装です。
- ここでは `確定した条件` と `まだ仮置きの条件` を分けて、試作のまま制度確定済みと誤解しないようにしています。

| 加算 | 現在の判定軸 | 確定した条件 | まだ仮置きの条件 | 否定的な見方 |
| --- | --- | --- | --- | --- |
| 医療・保育・教育機関等連携加算 | `児者` / `月区分` / `機関グループ` / `サービス判定区分` / `場所` / `行為` | `モニタリング月` または `計画作成月` で残す。`障害福祉サービス` は除外する。 | 機関グループの最終境界、`情報共有・会議・面談` の扱い、保育・教育系の最終線引き。 | いまは候補を落としすぎない寄りなので、制度上もっと早く除外すべき枝が残る可能性がある。 |
| 集中支援加算 | `児者` / `月区分` / `機関グループ` / `サービス判定区分` / `場所` / `行為` | `それ以外` 月で残す。後段では、`訪問` 枝だけ `同月2回以上の訪問` を確認する。 | 対象機関グループの最終範囲、`訪問` と `サービス提供場面確認` だけで足りるか。 | 訪問の回数要件は入れたが、まだ `訪問枝` と `サービス提供場面確認枝` を完全分離していないため、制度上の派生枝は今後も詰めが必要。 |
| モニタリング加算 | `児者` / `月区分` / `機関グループ` / `サービス判定区分` / `場所` / `行為` | `提供現場の訪問確認` で残す。`モニタリング月` に限定しない。後段では `同月1回まで` を確認する。 | 現行UIの `サービス提供場面確認` を、そのまま `提供現場訪問` とみなしてよいか。対象機関・対象サービスの最終境界。 | 旧資料では `モニタリング月でなくても可` となっているが、現行UIの行為ラベルはまだ制度用語と完全一致していない。ここを曖昧なまま広げると過剰候補になりやすい。 |
| 入院時情報連携加算 I | `児者` / `月区分` / `機関グループ` / `機関種別` / `サービス判定区分` / `場所` / `行為` | `病院` に `訪問` して `必要情報を提供` した場合に残す。後段では `同月1回まで` と `IIとの併算定不可` を確認する。 | 現行UIでは `訪問情報提供` を `外出先 + 情報共有` に寄せている。病院以外の医療機関をどこまで含めるか。 | `病院・訪看・薬局グループ` の中から `病院` だけへさらに絞るため、今回から `機関種別` 軸を追加している。ここを雑に広げると訪看や薬局まで拾ってしまう。 |
| 入院時情報連携加算 II | `児者` / `月区分` / `機関グループ` / `機関種別` / `サービス判定区分` / `場所` / `行為` | `病院` に `訪問以外の方法` で `必要情報を提供` した場合に残す。後段では `同月1回まで` と `Iとの併算定不可` を確認する。 | 現行UIでは `訪問以外情報提供` を `自事業所内 + 情報共有` に寄せている。病院以外の医療機関をどこまで含めるか。 | I と II を同名の1枝にまとめると請求単位が曖昧になるので、今回はあえて別候補にしている。逆に言うと、UI上の説明不足は今後の補強が必要。 |

| 退院・退所加算 | `児者` / `月区分` / `機関グループ` / `サービス判定区分` / `場所` / `行為` | `病院系` の `外出先` 対応で、`退院前面談` を要件にする。 | `計画作成月 / それ以外` の扱い、病院・訪看・薬局を同一枝に置く妥当性。 | この枝は7本の中で仮置きが最も多く、制度条件が固まるまでは過信しないほうがよい。 |

## 運用上の考え方

1. まずはこの7本を「残りすぎない / 落ちすぎない」状態まで寄せる。
2. 同じ条件が複数枝で繰り返し出るなら、枝固有条件ではなく共通軸へ引き上げる。
3. 7本が落ち着いたら、後段チェック (`回数制限`、`月上限`、`併算定不可`) を入れる。
今はこのうち、`月上限` と `集中支援加算の訪問枝の最低回数`、`入院時情報連携加算 I/II の相互併算定不可` まで入っている。
具体的な `どの加算とどの加算が併算定不可か` は、まだ広くは決めず、旧資料で明示されていた I / II の組み合わせだけを入れている。
4. そのあとで、`app.js` 直書きの試作枝を `addition` 系テーブルへ移す。

## 今回の確認対象

今回の時点で回帰確認したい最低条件は次です。

- `児童発達` のような `障害福祉サービス` では、医療・保育・教育機関等連携加算を残さない。
- `児童発達` のような `障害福祉サービス` かつ `外出先 + サービス提供場面確認` では、モニタリング加算を候補に残す。
- `病院` + `外出先` + `情報共有` では、入院時情報連携加算 I を候補に残す。
- `病院` + `自事業所内` + `情報共有` では、入院時情報連携加算 II を候補に残す。
- 担当者会議加算は `モニタリング月` かつ `担当者会議開催` でのみ残す。
- 集中支援加算は `それ以外` 月でのみ残す。
- 退院・退所加算は `病院系 + 外出先 + 退院前面談` がそろわないと残さない。
