- 試作候補が24本あるからといって、実際の加算種類が24種類しかない、という意味ではありません。

## 運用上の考え方

1. まずはこの24本を「残りすぎない / 落ちすぎない」状態まで寄せる。
2. 同じ条件が複数枝で繰り返し出るなら、枝固有条件ではなく共通軸へ引き上げる。
3. 24本が落ち着いたら、後段チェック (`回数制限`、`月上限`、`併算定不可`) を入れる。
今はこのうち、`月上限` と `グループごとの月上限`、`集中支援加算の訪問枝の最低回数`、`入院時情報連携加算 I/II の相互併算定不可`、`担当者会議加算` と `医保教（面談・会議）` の併算定不可まで入っている。
具体的な `どの加算とどの加算が併算定不可か` は、まだ広くは決めず、旧資料で明示されていた組み合わせだけを入れている。
4. そのあとで、`app.js` 直書きの試作枝を `addition` 系テーブルへ移す。

## 今回の確認対象

今回の時点で回帰確認したい最低条件は次です。

- `児童発達` のような `障害福祉サービス` では、医療・保育・教育機関等連携加算（情報共有 / 面談 / 会議）の各候補を残さない。
- `モニタリング月/計画作成月 + 病院系 + 外出先 + 通院同行` では、医療・保育・教育機関等連携加算（通院同行）を候補に残す。
- `医療・保育・教育機関等連携加算（面談）` と `（会議）` は、`病院・訪看・薬局グループ` では残さない。
- `児童発達` のような `障害福祉サービス` かつ `外出先 + サービス提供場面確認` では、モニタリング加算を候補に残す。
- `病院` + `外出先` + `情報共有` + `入院に当たっている` では、入院時情報連携加算 I を候補に残す。
- `病院` + `自事業所内` + `情報共有` + `入院に当たっている` では、入院時情報連携加算 II を候補に残す。
- `児` 対象 + `学校/保育/企業/就業生活支援センター` + `情報共有` では、保・教支援（情報共有）を候補に残す。
- `児` 対象 + `学校/保育/企業/就業生活支援センター` + `それ以外月 + 外出先 + 面談` では、保・教支援（訪問面接）を候補に残す。
- `児` 対象 + `学校/保育/企業/就業生活支援センター` + `それ以外月 + 会議` では、保・教支援（会議参加）を候補に残す。
- `者` 対象 + `ケアマネ事業所` + `情報共有` では、居宅連携（情報共有）を候補に残す。
- `者` 対象 + `ケアマネ事業所` + `それ以外月 + 外出先 + 面談` では、居宅連携（訪問面接）を候補に残す。
- `者` 対象 + `ケアマネ事業所` + `それ以外月 + 会議` では、居宅連携（会議参加）を候補に残す。
- `者` 対象 + `企業/障害者就業・生活支援センター` + `情報共有` では、居宅連携（就労）（情報共有）を候補に残す。
- `者` 対象 + `企業/障害者就業・生活支援センター` + `それ以外月 + 外出先 + 面談` では、居宅連携（就労）（訪問面接）を候補に残す。
- `者` 対象 + `企業/障害者就業・生活支援センター` + `それ以外月 + 会議` では、居宅連携（就労）（会議参加）を候補に残す。
- 医療・保育・教育機関等連携加算は、`病院・訪看・薬局グループ` で1件あっても、`福祉サービス等提供機関グループ` 側に同月記録がなければ別グループとして残せる。
- 担当者会議加算は `モニタリング月` かつ `担当者会議開催` でのみ残す。
- 集中支援加算（訪問）は `それ以外` 月かつ `訪問` でのみ残す。
- 集中支援加算（サービス提供場面確認）は `それ以外` 月かつ `サービス提供場面確認` でのみ残す。
- 集中支援加算（会議開催）は `それ以外` 月かつ `担当者会議開催` でのみ残す。
- 集中支援加算（会議参加）は `それ以外` 月かつ `会議` でのみ残す。
- 集中支援加算（通院同行）は `それ以外` 月かつ `病院系 + 通院同行` でのみ残す。
- 集中支援加算（情報共有）は `それ以外` 月かつ `情報共有` でのみ残す。
- 退院・退所加算は `病院系 + 外出先 + 退院前面談` がそろわないと残さない。
