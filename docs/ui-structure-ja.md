# v3 画面構成案

この資料は、`v3` の最初の画面骨格を固めるためのメモです。
ここで決めたいのは「何をどこに置くと相談員が迷いにくいか」であって、細かいUI装飾ではありません。

## 画面全体の形

- 上部バー
- 左ペイン: ボタンリンク
- 右ペイン: 画面内容

右ペインだけを差し替えて使う前提にします。
これなら日常操作の導線が固定され、判定画面と集計画面を行き来しても迷いにくくなります。

## 相談員向けの左ペイン案

日常的に使うものを上、たまに触るものを下に置きます。

| 並び順 | ボタン名 | 役割 | 備考 |
| --- | --- | --- | --- |
| 10 | 加算判定 | 1件の対応事実から加算候補を絞り込む | 毎日使う主画面 |
| 20 | 集計 | 請求前確認、月次確認、監査対応 | 毎日使う主画面 |
| 30 | 利用者 | 利用者マスタの確認と編集 | 相談員が直接触る |
| 40 | 機関 | 機関マスタの確認と編集 | 相談員が直接触る |
| 50 | サービス | サービスマスタの確認と編集 | 相談員が直接触る |

必要になっても、最初から左ペインに項目を増やしすぎないほうが安全です。
とくに `設定`, `履歴`, `管理`, `AI`, `帳票` のような項目を同列に増やすと、旧アプリと同じく迷いやすくなります。

## 上部バー案

| 項目 | 役割 |
| --- | --- |
| アプリ名 | いま何の画面を使っているかを常に示す |
| 対象月表示 | 月次業務中の取り違えを減らす |
| ログインアカウント名 | 誰の Google Workspace アカウントで入っているかを示す |
| 選択相談員名 | 今回の記録を誰の相談員名義で進めるかを示す |
| クイック検索 | 利用者名や機関名をすぐ探す |

上部バーは情報量を増やしすぎず、`今の文脈確認` を主目的にしたほうが良いです。

## 右ペインの主画面

### 1. 加算判定

右ペインの中では、さらに次の3段を基本にします。

| 領域 | 内容 |
| --- | --- |
| 上段 | 利用者、対象月、機関、サービス、選択相談員など現在の文脈 |
| 中段 | 今答える設問と選択肢 |
| 下段 | 残候補、回答済み内容、最終判定結果 |

補足:

- 1画面に全部の設問を並べるより、「今の1問」を中心に見せたほうが迷いにくいです。
- 参照項目は `ID保存・名称表示` にします。
- 補足説明や用語説明は持てるようにしつつ、表示方式は管理者設定で変えられる前提です。
- ログイン者と選択相談員が違う場合でも、画面上で見間違えないよう両方見える形にしたほうが安全です。

### 2. 集計フォーム

この画面は単なる一覧ではなく、`請求前確認` と `監査対応` の両方を担う前提にします。

| 領域 | 内容 |
| --- | --- |
| 上段 | 絞込条件フォーム |
| 中段 | 一覧結果 |
| 右寄せまたは下段 | 選択行の詳細 |

絞込条件の初期候補は、まず次を優先します。

| 項目 | 用途 |
| --- | --- |
| 対象月 | 月次請求の主軸 |
| 利用者 | 監査時の個別検索 |
| 加算 | どの加算を請求したか確認 |
| 判定状態 | 自動確定、要確認などの切り分け |
| 機関 | 機関別の確認 |
| 相談員 | 担当者別の確認 |

一覧結果では次をできるようにします。

- 列の表示順を入れ替える
- 表示列を増減する
- 並び順を変える
- その見え方を保存する
- 請求用と監査用で別の見え方を持つ
- 一覧の1行から判定詳細へすぐ飛ぶ

### 3. 利用者 / 機関 / サービス

3つのマスタは、同じ見え方の一覧 + 編集詳細に寄せるのが無難です。

| 項目 | 方針 |
| --- | --- |
| 一覧 | 名称を主表示、IDは補助表示 |
| 検索 | 名前で探せる |
| 詳細 | 右ペイン内で編集 |
| 削除 | 物理削除より無効化優先 |

## 否定的な見方も踏まえた注意点

| 懸念 | ありがちな失敗 | 今回の方針 |
| --- | --- | --- |
| 左ペインが増えすぎる | ボタンだらけでどこを押せばいいか分からない | 相談員向けはまず5本に絞る |
| 判定画面に情報を詰め込みすぎる | 設問も候補も説明も全部見えて混乱する | 主役は今の1問、補足は必要時だけ見る |
| 集計画面が何でも屋になる | フィルタも列も多すぎて表計算ソフト化する | 初期表示は請求と監査に必要な条件だけに絞る |
| マスタ画面が危険になる | 相談員が誤って制度設定をいじる | 相談員が触るのは `利用者`, `機関`, `サービス` のみに寄せる |

## 現時点の判断

最初に作るべきなのは、`加算判定` と `集計` がこの画面枠の中で素直に収まるかの確認です。
逆に、管理者向けの複雑な設定画面を先に広げるのはまだ早いです。
