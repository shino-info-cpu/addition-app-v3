# AppSheet データ取り込み可否メモ

## 対象ファイル

- 元データ: `J:\マイドライブ\加算アプリV2\加算入力アプリ２０２６NEW のコピー.xlsx`

## 先に結論

`v3` への取り込みは可能です。
ただし、`そのまま一括インポート` は無理で、`段階的インポート + 不足分レビュー` 前提になります。

理由は単純で、
AppSheet 側には `利用者`、`相談員`、`機関`、`サービス` はある一方で、
`利用者がどの機関のどのサービスを使っているか` を確定できるデータが足りないためです。

## 実データ確認結果

確認できた主なシートと件数は次のとおりです。

| シート | データ件数 | 主な列 | 取り込み評価 |
| --- | ---: | --- | --- |
| `利用者` | 398 | `利用者ID`, `利用者名`, `ふりがな`, `児者` | 直接可 |
| `相談員` | 5 | `相談員ID`, `相談員名`, `Email` | 直接可 |
| `機関` | 19 | `機関ID`, `機関名` | 直接可 |
| `サービス` | 31 | `サービスID`, `サービス名`, `児者`, `グループ`, `障害福祉別` | 正規化すれば可 |
| `加算記録` | 47 | `利用者ID`, `相談員ID`, `機関ID`, `児者区分`, `月区分`, `行動区分` など | 履歴として可 |
| `加算種別マスタ` | あり | `加算ID`, `加算名`, `内容区分` など | 参照元として可 |
| `加算条件` | あり | `児者区分`, `月区分`, `行動区分`, `機関区分` など | 参照元として可 |

## そのまま使えるもの

### 1. 利用者

そのまま `v3` の利用者系マスタに入れられます。

- `利用者ID`
- `利用者名`
- `ふりがな`
- `児者`

### 2. 相談員

そのまま `v3` の相談員マスタに入れられます。

- `相談員ID`
- `相談員名`
- `Email`

注意:
`所属機関ID` 列は見出しにありますが、実データは空でした。
なので AppSheet 現物からは所属機関は取れません。

### 3. 機関

そのまま `v3` の機関マスタに入れられます。

- `機関ID`
- `機関名`

### 4. サービス

旧メモより状況はよく、Excel には `サービス` シートがありました。
以下は取得できます。

- `サービスID`
- `サービス名`
- `児者`
- `グループ`
- `障害福祉別`

これは `v3` でいう

- サービス定義
- 対象区分
- サービスカテゴリ
- グループ分類

の初期値として使えます。

## そのままでは足りないもの

### 1. 利用者利用状況

ここが最大の不足です。

欲しいのは、

- 利用者が
- どの機関の
- どのサービスを
- どのグループで
- 使っているか

ですが、AppSheet 現物だけではこれを確定できません。

理由:

- `利用者` シートには `サービスID` 列がありますが、398件すべて空でした
- `加算記録` には `利用者ID` と `機関ID` はありますが、`サービスID` がありません

つまり、
`利用者 x 機関` の候補は見えても、
`利用者 x 機関 x サービス` までは自動確定できません。

### 2. 機関サービス

機関がどのサービスを正式に扱っているかの対応表がありません。
`サービス` シートはありますが、
`機関ごとの提供サービス一覧` は現物からは取れません。

### 3. グループの所属先

`サービス` シートには `グループ` 列があります。
ただしこれが

- サービス全体の分類なのか
- 機関ごとのグループなのか
- 利用者の所属グループなのか

は、現物だけでは断定できません。

なので `v3` では、最初は
`サービス側の分類ラベル`
として取り込み、必要なら後で正式な `グループ` マスタへ昇格させるのが安全です。

## v3 への取り込み方針

AppSheet データは 3 レーンに分けて扱うのが安全です。

### レーン1. 業務マスタ初期値

直接取り込む。

- 利用者
- 相談員
- 機関
- サービス

### レーン2. 判定ルール参照元

そのまま本番投入はせず、再設計の参照元として使う。

- 加算種別マスタ
- 加算条件
- 加算要件確認

### レーン3. 旧履歴

履歴として別テーブルへ入れるか、あるいは移行対象外にする。

- 加算記録
- 記録明細

## 可否判定

| 取り込み対象 | 可否 | 判定 |
| --- | --- | --- |
| 利用者 | 可 | 直接取り込み可 |
| 相談員 | 可 | 直接取り込み可 |
| 機関 | 可 | 直接取り込み可 |
| サービス | 可 | 列名正規化で可 |
| 機関サービス | 一部不可 | 別ソースか手確認が必要 |
| 利用者利用状況 | 一部不可 | 別ソースか手確認が必要 |
| 判定ルール | 参考可 | 再設計の入力資料として使える |
| 旧判定履歴 | 可 | 別レーンなら対応可能 |

## 実務上のおすすめ

1. まず `利用者`, `相談員`, `機関`, `サービス` だけを `v3` に初期投入する
2. `加算記録` から `利用者 x 機関` の実績候補表を作る
3. 不足している `サービス` と `グループ` をレビュー入力で補う
4. それを `利用者利用状況` の初期データにする
5. 判定ロジック系は AppSheet のまま持ち込まず、`v3` 決定木に合わせて再構成する

## 最終判断

AppSheet データの取り込みは、
`全部そのまま入るか` で言えば `不可` です。

ただし、
`v3 の初期データを作る材料として使えるか`
で言えば `十分に可` です。

なので方針としては、

- `自動で入るところは自動`
- `関係が欠けるところはレビュー表で補完`
- `ルール系は再設計して入れ直す`

が最も現実的です。
